🎨 Frontend Rendering

React Three Fiber provides a declarative interface to Three.js, enabling component-based 3D scene management with hot-reload support during development.

⚙️ Physics Simulation

Skyfield handles accurate ephemeris calculations. Kepler solver propagates orbital positions. Collision detection tests Earth-meteoroid stream intersections.

🔄 State Management

Zustand manages simulation time, mode selection, and UI state. Jotai atoms handle derived data (Earth position, ISS location). React Query caches API responses.

📡 Data Pipeline

NASA APIs provide ephemeris data. Space-Track supplies live TLE data. Local IndexedDB caches static catalogs (meteor showers, comet orbits).

Meteor Shower "Why Simulator" Flow

1
User Input: Time scrubber changes to August 11
2
State Update: useOrbitalTime hook updates Zustand store
3
Physics Calculation: useEarthPosition recomputes Earth's position/rotation for new date
4
Collision Test: Check if Earth is within meteoroid stream radius (simple distance test)
5
Shader Update: Set uniform isActive = true if intersecting, adjust particle glow
6
Telemetry Update: Sidebar shows "Earth is intersecting: YES" + relative velocity
7
Render: WebGL draws updated scene with highlighted particles

ISS Real-Time Tracker Flow

1
App Mount: React Query fetches latest ISS TLE from Space-Track API
2
Parse TLE: Extract orbital elements (inclination, RAAN, eccentricity, etc.)
3
Every Frame: SGP4 propagator computes ISS position at current time
4
Ground Track: Project ISS position onto Earth surface, show in telemetry
5
Next Pass:** Calculate when ISS will be visible from user's location

Why This Stack?

Performance: Instanced rendering + Web Workers allow smooth 500K+ particle updates
Accuracy: Skyfield provides peer-reviewed astronomical calculations
Developer Experience: React Three Fiber + hot-reload makes iteration fast
Bundle Size: Vite tree-shaking keeps gzipped bundle under 12MB

Alternatives Considered

Babylon.js - Equally capable, slightly heavier
Cesium.js - Great for maps, overkill for solar system
Redux - Too heavy for our state patterns
Python (Flask/FastAPI) - Could host physics on backend, but would add latency

Kepler Solver (6-DOF Propagator)

Input: Six orbital elements + Julian Date
Process: Newton-Raphson iteration to solve Kepler's equation
Output: 3D position (x,y,z) + velocity vector
Accuracy: ±1 km for 2-year propagation

Earth Rotation (GMST + Axial Tilt)

GMST: Greenwich Mean Sidereal Time rotates inertial frame to fixed frame
Axial Tilt: 23.44° inclination causes seasons
Precession: Slow drift of rotation axis over 26,000 years
Effect: Changes radiant point altitude by latitude/date

Meteoroid Stream Intersection

Stream Model: Orbital tube following comet's path
Test: Sphere-point distance: is Earth within stream radius?
Visual Feedback: Shader glow when intersecting
Insight: Shows exactly when Earth crosses debris field

Two-Line Element (TLE) Propagation

Format: Compact orbital definition for LEO satellites
Algorithm: SGP4/SDP4 (analytic perturbation model)
ISS Updates: New TLE daily from Space-Track
Use: Calculate ISS position for next pass predictions
